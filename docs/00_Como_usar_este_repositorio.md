# Cómo usar este repositorio

Este material está diseñado como repositorio de información y práctica técnica. No es solo una lista de pasos de clase; es una base de estudio que el estudiante puede consultar antes, durante y después de cada sesión.

## Flujo sugerido por semana

1. Leer el README semanal completo.
2. Revisar el mapa conceptual y los diagramas.
3. Identificar los conceptos que se aplican a la aplicación de referencia.
4. Realizar la práctica técnica sugerida.
5. Desarrollar la tarea desde cero en un repositorio propio.
6. Subir la solución a GitHub.
7. Compartir el enlace en Classroom mientras se habilita el sistema propio.

## Convención de repositorios del estudiante

Nombre recomendado:

```text
curso-arquitectura-cloudops-nombre-apellido
```

Estructura recomendada:

```text
.
├── README.md
├── semana-01/
├── semana-02/
├── semana-03/
└── evidencias/
```

Cada entrega debe incluir un README con:

- Objetivo de la práctica.
- Diagrama de arquitectura o flujo.
- Instrucciones de ejecución.
- Decisiones técnicas.
- Capturas o evidencias.
- Retos encontrados.
