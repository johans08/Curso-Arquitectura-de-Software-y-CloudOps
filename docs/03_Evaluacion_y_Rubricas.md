# Evaluación y rúbricas

## Distribución general

| Componente | Ponderación |
|---|---:|
| Prácticas semanales | 40% |
| Exámenes modulares | 30% |
| Proyecto integrador | 30% |

## Rúbrica para prácticas semanales

| Criterio | Excelente | Aceptable | Requiere mejora |
|---|---|---|---|
| Funcionalidad | Cumple todos los requisitos y casos esperados. | Cumple lo principal con errores menores. | No ejecuta o no cumple el objetivo. |
| Arquitectura | Separación clara, bajo acoplamiento y decisiones justificadas. | Separación parcial con algunos acoplamientos. | Código mezclado sin criterio arquitectónico. |
| Documentación | README completo con diagrama, ejecución y decisiones. | README básico con instrucciones mínimas. | Sin explicación suficiente. |
| Git | Commits claros y evolución visible. | Pocos commits pero trazables. | Sin historial útil. |
| Calidad técnica | Validaciones, manejo de errores y pruebas básicas. | Implementación correcta pero incompleta. | Falta control de errores o prácticas inseguras. |
