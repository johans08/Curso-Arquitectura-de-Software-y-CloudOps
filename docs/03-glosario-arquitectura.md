# Glosario de arquitectura

## API

Contrato HTTP que expone capacidades de un sistema a otros clientes o servicios.

## Application Service

Clase que representa un caso de uso. Coordina entidades, repositorios, transacciones y servicios externos.

## Blazor

Framework de frontend de .NET para construir interfaces web usando C# y Razor Components.

## Clean Code

Conjunto de prácticas orientadas a hacer el código comprensible, mantenible y seguro de modificar.

## DTO

Objeto de transferencia de datos. Evita exponer directamente entidades internas.

## Entity Framework Core

ORM de .NET que permite mapear objetos C# a tablas relacionales.

## JWT

Token firmado que transporta claims para autenticar y autorizar solicitudes.

## Monolito modular

Aplicación desplegada como una sola unidad, pero organizada internamente en módulos con límites claros.

## Microservicio

Servicio pequeño, autónomo, desplegable de forma independiente y dueño de sus datos.

## Outbox Pattern

Patrón que guarda eventos en la misma transacción que los cambios de negocio para procesarlos luego de forma confiable.

## SOLID

Cinco principios de diseño orientado a objetos para reducir acoplamiento y mejorar mantenibilidad.

## SQL Server

Motor relacional usado en el módulo para persistencia, consultas, transacciones e integridad de datos.
