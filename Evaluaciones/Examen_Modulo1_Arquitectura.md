# Examen modular 1: Arquitectura de Software y Patrones

## Modalidad

Prueba técnica individual con análisis y práctica.

## Parte A: Conceptual

1. Explique SRP y DIP con un ejemplo de ASP.NET Core.
2. ¿Cuándo usaría un monolito modular en vez de microservicios?
3. Diferencie Factory, Adapter y Decorator con ejemplos.
4. Explique por qué OpenAPI es un contrato y no solo una página de Swagger.
5. Explique el problema que resuelve el Outbox Pattern.

## Parte B: Práctica

Diseñe un módulo de inventario con:

- API documentada.
- Servicio de aplicación.
- Persistencia SQL Server.
- Evento de inventario actualizado en Outbox.
- Protección JWT para endpoints de escritura.
