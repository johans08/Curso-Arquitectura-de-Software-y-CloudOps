# Guía de contribución y entregas

## Flujo sugerido

1. Crear una rama por semana.
2. Resolver laboratorio y tarea.
3. Documentar evidencias en el README de la entrega.
4. Abrir Pull Request para revisión.

## Estándar mínimo de código

- Nombres claros.
- Métodos pequeños.
- Separación de responsabilidades.
- Validaciones explícitas.
- Manejo básico de errores.
- Endpoints probados con ejemplos.

## Formato de Pull Request

```md
## Semana
Semana X - Tema

## Qué se implementó
- ...

## Cómo probar
```bash
dotnet run
```

## Evidencias
- Capturas o ejemplos JSON

## Reflexión técnica
- Decisiones tomadas
- Trade-offs
- Mejoras futuras
```
